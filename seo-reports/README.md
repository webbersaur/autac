# seo-reports

Tooling for the autacusa.com SEO recovery program (WP → static migration, Apr 2026).

| Script | Needs GSC creds? | What it does |
|---|---|---|
| `refresh_report.py` | yes | Weekly GSC pull → `autac-keyword-report.html` |
| `track_recovery.py` | yes | Page-level recovery scorecard for the 30 restored/moved URLs → `recovery-scorecard.html` |
| `restore_posts.py` | no (uses a Bluehost SQL dump) | Rebuilds deleted WP posts as static HTML |

## Google Search Console credentials

`refresh_report.py` and `track_recovery.py` both authenticate to the GSC API
using an OAuth **refresh token** read from `~/.env`. Without it they exit (or
return all-zero rows).

### 1. Create `~/.env` with three keys

The scripts call `Path.home() / ".env"` and read these exact names:

```sh
GOOGLE_CLIENT_ID=xxxxxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxxxxxxx
GOOGLE_REFRESH_TOKEN=1//xxxxxxxx
```

Values are stripped of surrounding quotes, so quoted or unquoted both work.
This file is **not** in the repo and must never be committed — it lives in the
home directory, outside the working tree, on purpose.

### 2. Where the values come from

They are a Google Cloud OAuth 2.0 **Desktop/Web client** plus a refresh token
minted against the read-only Search Console scope:

```
https://www.googleapis.com/auth/webmasters.readonly
```

- `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` — from the OAuth client in the
  Google Cloud project (APIs & Services → Credentials). The **Search Console
  API** must be enabled on that project.
- `GOOGLE_REFRESH_TOKEN` — obtained once by running the consent flow for the
  scope above and saving the returned refresh token. These scripts only ever
  *use* the refresh token; they never run the consent flow themselves. The
  token was originally generated by the companion `mcp-gsc` project's auth flow.

The Google account that grants consent must have at least **read access to the
GSC property** named in `SITE_URL` (see below).

### 3. The GSC property string must match

Both scripts query:

```python
SITE_URL = "https://autacusa.com/"   # non-www URL-prefix property
```

This must be an exact GSC property the token can read. If GSC only has the
`www.` URL-prefix property (or a Domain property), change `SITE_URL` to match —
otherwise `searchanalytics().query()` returns no rows.

> Note: URL matching against the tracked list is www/non-www agnostic
> (`norm_url()` strips scheme + `www.` + trailing slash), so the *property*
> string is the only place the www form matters.

## Python environment

The scripts import `google-auth` and `google-api-python-client`:

```sh
python3 -m venv .venv
source .venv/bin/activate
pip install google-auth google-api-python-client
```

Then run from this directory:

```sh
python track_recovery.py      # recovery scorecard
python refresh_report.py      # keyword report
```

Outputs are written next to the scripts: a date-stamped archive plus a
`*-latest` copy (`recovery-scorecard.html`, `autac-keyword-report.html`).
